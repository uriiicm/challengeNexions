Este usuario tiene asignado el perfil de {{perfil | uppercase}}, el cual tiene permisos para dar de alta Formularios

En dicha sección podras dar de alta formularios, registrando la siguiente información:

  • Nombre del formulario
  • Clave
  • Campos
    • Label
    • Tipo de campo

Este usuario tiene asignado el perfil de {{perfil | uppercase}}, el cual tiene permisos para consultar todos los Formularios existentes y así poder responder el que necesite.

Este usuario tambien tiene disponible la sección de Mis Formularios en donde podrá visualizar los formularios que ha llenado.

Las dos secciones cuentan con un buscador que le permitira al usuario filtrar los formularios por medio de su cláve.

Es esta aplicación se utilizó lazy loading y se optó por hacer uso del módulo PreloadAllModules ya que considero que son pocos componentes los que se cargan en cada módulo, tomando en cuenta esto... pienso que podría ser de gran ayuda el tener precargadas ya las rutas para cuando el usuario las necesite.

Si la aplicación fuera más grande, lo más probable es que hubiera optado por QuicklinkStrategy ya que no sería bueno precargar todas las rutas, si no cargar solo las asociadas a la página en la que se encuentre el usuario , para así poder gestionar mejor la memoria de la aplicaciíon y no saturar la red.
